Axe Public Intent API

Empower your Android apps and games with Discord Rich Presence. Axe provides a seamless bridge between your application and Discord, handling the complex gateway logic so you can focus on your product.

Setup / Prerequisites

1. Custom Permission

Declare the following permission in your AndroidManifest.xml to authorize your app to communicate with Axe.

<uses-permission android:name="com.my.axe.PERMISSION_RPC" />

2. Package Visibility

For apps targeting Android 11+, include Axe in your queries block:

<queries>
    <package android:name="com.my.axe" />
</queries>

Available Intent Actions

Action Description
com.my.axe.UPDATE_PRESENCE Updates or starts a Discord presence.
com.my.axe.CLEAR_PRESENCE Stops and clears the current presence.

Supported Extras

Use these extras with the UPDATE_PRESENCE action.

Key Type Description
activity_nameStringPrimary activity title
detailsStringSecond line of text
stateStringThird line of text
large_imageStringPublic URL for large image
small_imageStringPublic URL for small image
timestamp_startLongStart time (ms)
activity_typeStringplaying, listening, watching...
source_packageStringYour package name (required)

Complete Kotlin Example

fun updateAxePresence(context: Context) {
    val intent = Intent("com.my.axe.UPDATE_PRESENCE").apply {
        `package` = "com.my.axe"
        putExtra("activity_name", "Kingdom of Kotlin")
        putExtra("details", "Battling the NullPointerException")
        putExtra("state", "Level 99 Mage")
        putExtra("large_image", "https://example.com/assets/world_map.png")
        putExtra("timestamp_start", System.currentTimeMillis())
        putExtra("activity_type", "playing")
        putExtra("source_package", context.packageName)
    }
    context.sendBroadcast(intent)
}

Java Example

public void updateDiscordPresence(Context context) {
    Intent intent = new Intent("com.my.axe.UPDATE_PRESENCE");
    intent.setPackage("com.my.axe");
    intent.putExtra("activity_name", "Java Legends");
    intent.putExtra("details", "Managing GC Overhead");
    intent.putExtra("timestamp_start", System.currentTimeMillis());
    intent.putExtra("activity_type", "playing");
    intent.putExtra("source_package", context.getPackageName());
    context.sendBroadcast(intent);
}

Best Practices & Tips

  • Rate Limiting: Minimum 15 seconds between updates.
  • Image URLs: Must be direct public links (not redirects).
  • Cleanup: Always call CLEAR_PRESENCE when your activity finishes.

Clear Presence Example

val intent = Intent("com.my.axe.CLEAR_PRESENCE").apply {
    `package` = "com.my.axe"
}
context.sendBroadcast(intent)
